# Demo Checklist

## Phase 1

- A plain text math problem can be entered from the terminal.
- The app prints a final answer.
- The app prints a step-by-step explanation.
- The run command is clear enough for another attendee to try.

## Phase 2

- The terminal output makes the subagents visible.
- The review agent returns an explicit status.
- The final explanation is gated on approval.
- There is at least one visible fallback or revise path.

## Phase 3

- A user can type a problem in the UI and submit it.
- A user can upload a problem image.
- The screen shows processing states.
- The screen shows a final answer and explanation when approved.
- The screen shows a friendly failure card when review fails or image parsing is unclear.

## Final Workshop Demo

- One normal text problem
- One slightly tricky text problem
- One image-based problem
- One failure or fallback case
